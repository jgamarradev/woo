¿Cómo asegurarías que tu plugin no cause conflictos con otros plugins o temas instalados?

Existen buenas practicas establecidas para evitar que esto suceda, se pueden encontrar en el Plugin Handbook de Wordpress.org. Personalmente sigo los siguientes:
  • 1.- Es común la dependencia de plugins en el desarrollo de plugins ya sea para extender funcionalidades pero es recomendable evitarlas en lo posible, una de las mas comunes es la dependencia de WooCommerce.
  • 2.- Suelo usar hooks y filtros para realizar modificaciones a temas y plugins para no modificar codigo en el core de los mismos.
  • 3.- Uso encolado de scripts y de estilos para componentes base.
  • 4.- Uso de condicionales para cargar ciertos estilos, funciones y scripts para paginas especificas, wordpress provee de funciones para el manejo de esta practica.

  • 6.- Uso de los hooks register_uninstall_hook y register_deactivation_hook para limpiar datos almacenados por nuestro plugin.

Explica la diferencia entre acciones y filtros en WordPress. Proporciona un ejemplo de cada uno.

La diferencia es que las acciones permiten ejecutar codigo en un momento y lugar especifico sin modificar los valores, en cambio los filtros modifican los datos antes de mostrarlos o guardarlos.

¿Cómo manejarías la seguridad en una tienda WooCommerce? Menciona al menos tres prácticas recomendadas.

  • 1.-Toda tienda o sitio web debe tener como base principal un buen hosting, todo esto debe contar como una buena practica primordial. Un buen hosting es aquel que provee SSL, Backups, monitoreo de ataques, un firewall en servidor, un buen soporte 24/7 y que permita actualizar php y mysql.
  • 2.El factor humano es el mas vulnerable en la seguridad de cualquier aplicación, el uso de contraseñas fuertes en las cuentas ya se de administradores o clientes es importante.
  • 3.-Uso de plugins de seguridad que tengan monitoreo y bloqueo de ataques de fuerza bruta
